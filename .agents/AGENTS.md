# Правила проекта / Project Rules

- **Язык общения и перевода**: Все ответы, пояснения, тексты интерфейса Antigravity, описания расширений (extensions), плагинов, навыков и системных сообщений должны полностью и всегда переводиться на русский язык.

- **Автоматическое решение MCP ошибок**: При появлении ошибок `MCP Error` (`notebooks`, `visualization`) запускать навык `mcp-config-repair` или удалять сбоящие прокси-серверы из `mcp_config.json`.

- **Правило пуша на GitHub**: Пушить изменения в репозиторий GitHub СТРОГО только по прямому указанию пользователя или после его явного согласия на вопрос о пуше. Не делать авто-пуш при каждом изменении файла.

- **Версионирование на GitHub (Tags & Releases)**: Для версионирования кода и возможности быстрого отката назад при ошибках использовать стандартный профессиональный механизм **Git-тегов и релизов (Git Tags)** (например, `v1.0-text`, `v1.1-voice`, `v1.2-layout-fix`). Отмечать каждый стабильный этап версией (тегом), а не создавать отдельные дублирующие папки для каждой версии.
